# OpenSCAD Libraries Reference

A curated catalogue of third-party OpenSCAD libraries (sourced from
https://openscad.org/libraries.html, snapshot 2026-05-18). Planning and
implementation tasks should scan this list before writing new geometry from
scratch — many common patterns (rounded corners, threads, hinged enclosures,
hardware specs) are already solved here.

As of this writing, no external library is used in the repo. All `.scad` files
are self-contained. Adopting a library is a deliberate choice that should be
proposed in the issue or plan, not assumed by an implementer.

## Before adopting a library

Spell out each of the following in the plan before proposing a library:

- **CI tool chain**: The runner already has OpenSCAD, ImageMagick, ADMesh,
  qrencode, zip, xvfb, Python 3, and AWS CLI (see `docs/OVERVIEW.md`
  "Configuration"). Pure-`.scad` libraries need only to be vendored under a
  top-level `lib/` directory and referenced via `include`/`use`; no new install
  step is required.
- **Library detection heuristics**: CI skips underscore-prefixed `_*.scad`
  files by convention, detects "top level object is empty" from OpenSCAD log
  output, and falls back to an empty-STL heuristic (see OVERVIEW "Library vs.
  Renderable Files"). Vendored library files without the `_` prefix may still
  be skipped by the empty-output heuristic, but it is safer to either (a) place
  them under a `lib/` directory excluded from rendering, or (b) rename to
  `_*.scad`. The plan must pick one.
- **Dependency graphs**: `dependency-graph.md` files are generated by
  `scripts/scad-dep-graph.sh` and CI verifies they stay current. Any new
  `include <lib/foo.scad>` will appear in the graph and must regenerate
  cleanly.
- **Pinned OpenSCAD version**: `.openscad-version` stores the expected version
  baseline; CI warns on drift. If a library requires a newer OpenSCAD than the
  pinned version, the plan must include a version bump.
- **License compatibility**: The repo default license applies unless
  `meta.json.license` overrides it. Check that the library's license is
  compatible before adopting.
- **Fit regression risk**: Substituting library geometry for hand-rolled
  geometry on already-printed-and-verified parts (e.g.,
  `power-workshop/_connection.scad`) risks sub-millimetre STL changes (different
  `$fn` defaults, hull-vs-extrude variations). Call out fit-regression risk
  explicitly when proposing any such substitution.

## General-Purpose Libraries

### BOSL — https://github.com/revarbat/BOSL

**What it is:** An older general-purpose toolkit of shapes, transforms, and
helper modules for OpenSCAD; largely superseded by BOSL2.

**When to reach for it in this repo:** No obvious current fit — listed for
completeness. Prefer BOSL2.

---

### BOSL2 — https://github.com/BelfrySCAD/BOSL2/

**What it is:** A comprehensive shapes-and-helpers library covering rounded
primitives, threading, screws, hex and polygon helpers, snap-fit joiners, and
much more.

**When to reach for it in this repo:** The single highest-value library to
evaluate first for any new model. Rounded primitives could replace hand-rolled
`hull()` bevels in `power-workshop/_connection.scad`; threading, screws, and
snap-fit joiners are built in for any future hardware-adjacent model.

---

### dotSCAD — https://github.com/JustinSDK/dotSCAD

**What it is:** A math-oriented modelling helper library covering splines,
fractals, polyhedra, and other computational geometry utilities.

**When to reach for it in this repo:** No obvious current fit — useful if a
future model needs parametric curves or generative geometry.

---

### NopSCADlib — https://github.com/nophead/NopSCADlib

**What it is:** A parts library for 3D printers and electronics enclosures,
including fans, PSUs, screws, nuts, profile extrusions, and more.

**When to reach for it in this repo:** Any future workshop fixture,
electronics-adjacent model, or enclosure for hardware — the pre-modelled
mechanical parts would reduce from-scratch work.

---

### UB.scad — https://github.com/UBaer21/UB.scad

**What it is:** A full workflow toolkit providing object modifiers, view
helpers, and mechanical parts for OpenSCAD.

**When to reach for it in this repo:** No specific current fit — general
utilities to survey if a model's workflow becomes complex.

---

### Functional OpenSCAD — https://github.com/thehans/FunctionalOpenSCAD

**What it is:** Implements OpenSCAD primitives as pure functions returning
point and face data rather than geometry objects, enabling procedural mesh
construction.

**When to reach for it in this repo:** Advanced use only — relevant if a
model needs procedural mesh construction beyond what CSG primitives allow.
No current fit.

---

### Constructive Library — https://github.com/solidboredom/constructive

**What it is:** Extends OpenSCAD syntax for mechanical assemblies, reducing
explicit math in multi-part composition.

**When to reach for it in this repo:** Could simplify multi-part composition
in `adjustable-bracket` and `power-workshop` attachments; worth evaluating
before the next multi-part model.

---

### StoneAgeLib — https://github.com/Stone-Age-Sculptor/StoneAgeLib

**What it is:** A grab-bag collection of 3D printing scripts covering various
utilities and geometric helpers.

**When to reach for it in this repo:** No specific fit — scan for useful
one-offs only; not a library to depend on as a whole.

---

### BOLTS — https://github.com/boltsparts/BOLTS

**What it is:** An open library of technical specifications for standard
fasteners (bolts, nuts, washers, screws) with named-part lookups.

**When to reach for it in this repo:** Could replace hardcoded fastener
dimensions in `adjustable-bracket` (M5 bolt, nut, washers) and `blast-gate`
(M4/#8 clearance holes); `meta.json.hardware` part IDs could reference BOLTS
part names for traceability.

---

### Asset Collection — https://github.com/AngeloNicoli/OpenSCAD-Snippet

**What it is:** A mixed collection of OpenSCAD snippets covering mechanical
parts, furniture, and other objects.

**When to reach for it in this repo:** Scan only; no stable library API to
depend on.

---

### Board Game Toolkit — https://github.com/pinkfish/openscad_boardgame_toolkit

**What it is:** A toolkit specifically for generating box inserts for board
games, with compartments, lids, and organiser layouts.

**When to reach for it in this repo:** No current fit — relevant only if
board-game inserts are added to this repo.

---

## Single-Topic Libraries

### Round Anything — https://github.com/Irev-Dev/Round-Anything

**What it is:** A focused library of 2D and 3D rounding utilities for adding
fillets and chamfers to OpenSCAD geometry.

**When to reach for it in this repo:** Directly applicable to the
rounded-corner techniques in `power-workshop/_connection.scad`
(`offset(r)/offset(delta)` shaft profile, `hull()` bevels) and any future
part needing fillets. A lighter-weight alternative to BOSL2 if rounding is
the only need.

---

### Mark's Enclosure Helper — https://github.com/sbambach/MarksEnclosureHelper

**What it is:** Generates two-piece hinged enclosures with snap, magnet, or
screw closures and rounded corners.

**When to reach for it in this repo:** No current fit — useful for future
hinged-box models.

---

### funcutils — https://github.com/thehans/funcutils

**What it is:** Higher-order function utilities (map, fold, etc.) implemented
using OpenSCAD function literals.

**When to reach for it in this repo:** Companion to Functional OpenSCAD;
advanced and niche. No current fit.

---

### threads.scad — https://github.com/rcolyer/threads-scad

**What it is:** Generates accurate metric threads, nuts, and bolts in
OpenSCAD.

**When to reach for it in this repo:** The most direct fit when any threaded
model is needed (e.g., a screw-on cap or threaded vacuum-hose adapter); no
current fit.

---

### Smooth Primitives Library — https://github.com/rcolyer/smooth-prim

**What it is:** Provides OpenSCAD primitives (cylinders, cubes, spheres) with
automatically rounded edges.

**When to reach for it in this repo:** Overlaps with Round Anything in problem
space — pick one if rounding becomes pervasive across the repo.

---

### Function Plotting Library — https://github.com/rcolyer/plot-function

**What it is:** Plots mathematical functions as 3D surfaces in OpenSCAD.

**When to reach for it in this repo:** No fit — this repo models printable
parts, not mathematical visualisations.

---

### ClosePoints Library — https://github.com/rcolyer/closepoints

**What it is:** Builds polyhedra from arrays of points, useful for swept or
lofted profiles.

**When to reach for it in this repo:** Advanced alternative to the
`hull()`-of-discs taper technique used in `vacuum-hose/adapter.scad` and
`vacuum-hose/reducer.scad`.

---

### Tray Library — https://github.com/sofian/openscad-tray

**What it is:** Generates subdivided storage trays with configurable
compartments, walls, and lids.

**When to reach for it in this repo:** Worth evaluating for `sink-tray`
variants or new tray/organiser models.

---

### YAPP_Box — https://github.com/mrWheel/YAPP_Box

**What it is:** Generates electronics project boxes driven by PCB dimensions,
with configurable cutouts, mounts, and lids.

**When to reach for it in this repo:** No current fit — relevant for future
electronics enclosure models.

---

### STEMFIE Parts Library — https://github.com/Cantareus/Stemfie_OpenSCAD

**What it is:** Parametric parts for the STEMFIE open construction system
(beams, connectors, fasteners).

**When to reach for it in this repo:** No current fit — relevant only if
STEMFIE-compatible parts are added.

---

### Catch'n'Hole — https://github.com/mmalecki/catchnhole

**What it is:** Ergonomic modules for nut catches, screw holes, and
countersinks sized to standard hardware.

**When to reach for it in this repo:** Directly applicable to
`adjustable-bracket` (M5 nut catch) and `blast-gate` (M4/#8 clearance holes
in the mounting plate). High-priority candidate the next time a part involves
captured fasteners.

---

### Pathbuilder — https://github.com/dinther/pathbuilder

**What it is:** Constructs 2D paths with fillets and chamfers using an
SVG-like syntax before extrusion.

**When to reach for it in this repo:** Niche — useful for complex 2D profiles
that would otherwise require manual point arrays.

---

### SCON — https://github.com/wmacevoy/openscad-scon/

**What it is:** Provides JSON-like structured configuration data for OpenSCAD,
enabling external parameter files.

**When to reach for it in this repo:** Only helpful if cross-file parameter
centralisation is wanted; the current per-project `_*.scad` library pattern
already addresses this. No current fit.

---

### Altair's 2D Library (A2D) — https://github.com/ridercz/A2D

**What it is:** A collection of 2D geometry utilities for OpenSCAD, covering
polygons, text layout, and 2D operations.

**When to reach for it in this repo:** Niche — useful for 2D-heavy parts such
as badges or text plates; no current fit.

---

### Doll House — https://codeberg.org/adrien-delhorme/openscad-doll-house

**What it is:** A library of parametric wooden doll-house parts (walls,
floors, roofs, furniture outlines).

**When to reach for it in this repo:** No current fit — listed for
completeness.

---

### Dimensions — https://codeberg.org/adrien-delhorme/openscad-new-dimensions

**What it is:** Draws dimension annotation arrows and labels inside an OpenSCAD
model, similar to engineering drawing callouts.

**When to reach for it in this repo:** Could complement `CONNECTION_SPEC.md`
and `DRILL_SOCKET_SPEC.md` if generated dimension drawings are ever wanted —
an annotated `.scad` file could replace or supplement the text spec files.

---

## Keeping this list current

This catalogue is a manual snapshot of https://openscad.org/libraries.html as
of 2026-05-18. There is no CI check that detects drift. When working on this
repo and encountering a library not listed here, add it via a small follow-up
issue.
