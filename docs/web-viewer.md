# Web Viewer

Detailed reference for `index.html`, `embed.html`, and the generated
standalone/OEmbed outputs. See [OVERVIEW.md](OVERVIEW.md) for how these fit
into the wider repo.

## Web Viewer (index.html)

A single-page application (no build tools, no framework) with inline CSS and
ES module JS. Pulls Three.js from a CDN via import map.

### Core Functionality

- Fetches `models.json` (generated by CI) listing all STL files grouped by
  project, with optional zip bundle references, metadata, and print time
  estimates
- Renders each model in an interactive Three.js canvas with orbit controls
- Provides download links for individual STL files and a link to the
  standalone HTML viewer per model
- For projects with 2+ models, shows a "Download All" button linking to a
  CI-generated zip bundle of all STLs in the project
- For every project, shows a "Download Source" button linking to a
  CI-generated zip of the project's git-tracked source files
- Supports fullscreen mode per model card
- All project sections are collapsed by default on both the main site and PR
  previews, avoiding spinning up every WebGL context at once. In PR previews,
  `changed.json` auto-expands only the modified projects. A `#project/model`
  hash expands the target section via `navigateToHash()` in both environments
- Uses a `__BUILD_HASH__` placeholder replaced at build time with the commit
  SHA for cache busting
- Includes Open Graph, Twitter Card meta tags, and Schema.org JSON-LD
  structured data (`CollectionPage` with `3DModel` items) for SEO — the
  JSON-LD is injected at build time via a `<!-- __STRUCTURED_DATA__ -->`
  placeholder
- OEmbed `<link rel="alternate">` discovery tags are injected at build time
  via a `<!-- __OEMBED_LINKS__ -->` placeholder
- Head includes `<link rel="canonical">`, `<meta name="theme-color">`,
  `<meta name="color-scheme" content="dark">`, SVG favicon links, and
  `<link rel="manifest">` pointing to `site.webmanifest`

### Security Convention

All dynamic content in `index.html` and `embed.html` that interpolates model
names, filenames, or other data into the DOM uses the DOM API
(`createElement` / `textContent` / `setAttribute`) rather than `innerHTML`.
`innerHTML` is only used for static SVG icons and the gesture-hint overlay
(which contains no user-controlled data). This prevents XSS from
adversarially-named STL files or unexpected `models.json` content.
As a defense-in-depth layer, CI refuses to render any `.scad` file whose
basename contains characters outside `[A-Za-z0-9._ -]` (see step 6 in
[ci-pipeline.md](ci-pipeline.md)).

For **standalone HTML viewers**, `scripts/generate-standalone.py` embeds
filament color data inside a `<script type="module">` block. The
`_load_filament_colors_js()` function uses `json.dumps(name)` to produce a
valid double-quoted JS string literal (handling `"`, `\`, control characters,
and other special sequences), then additionally replaces `<`, `>`, and `&`
with unicode escapes (`\u003c`, `\u003e`, `\u0026`). This prevents a
`</script>` sequence in a color name from terminating the surrounding script
block — an injection vector that would otherwise allow same-origin script
execution in PR preview deployments. Covered by
`scripts/test_generate_standalone.py`.

### Print Time Estimates

Each model card displays a rough print-time badge (e.g., `~25m`, `~1h 30m`)
computed from ADMesh's bounding-box height and volume during CI. The heuristic
uses 0.2mm layer height and 50mm/s speed as conservative defaults. For
near-flat models (`bb_z < 0.5`), a simpler volume-based fallback is used.
The `formatTime()` function formats minutes into human-readable strings. The
badge is omitted if the estimate is unavailable (graceful degradation).

### Lazy Loading

3D viewers are not created on page load. Instead, each card displays a PNG
thumbnail placeholder (same basename as the STL, generated by CI). Thumbnail
`<img>` elements have `loading="lazy"` and `decoding="async"` set at creation
time to defer off-screen image decoding. Viewers are initialized via
`IntersectionObserver` (`rootMargin: '200px'`) as cards scroll into view. A second `IntersectionObserver` pauses/resumes the
`requestAnimationFrame` loop for off-screen initialized viewers to save GPU
work. The `activateCard()` function manages the placeholder-to-canvas swap.

A `cardState` Map keyed by card DOM element tracks per-card state: canvas,
placeholder, STL URL, viewer instance, initialization flag, selected color,
and touch hint overlay reference.

### Filament Color Picker

Each model card has a row of 8 circular color swatches (Blue, White, Black,
Grey, Red, Green, Yellow, Orange). The palette is defined in
`filament-colors.json` — a single source of truth shared by `index.html`
(fetched at runtime) and `generate-standalone.py` (injected at build time).
Clicking a swatch updates the Three.js `MeshPhongMaterial` color. The
material is created before the STL loads, so color can be changed before or
after loading. The selected color is stored in `cardState` and applied when
the viewer initializes (if changed before activation). White uses `#f5f5f5`
(not pure white) and Black uses `#2a2a2a` (not pure black) to preserve
surface detail under the Phong lighting model.

### Composite Multi-Colour Assembly Previews

STL export is monochrome, so a single merged mesh can't reproduce an
OpenSCAD `color()` preview's per-part colours — and for `nz-ski-fields`, a
full-resolution merged assembly mesh was heavy enough to freeze a CI runner
(issue #272) and crash browsers loading the gallery page. Instead, a
project's `meta.json` can declare an `assembly` object (schema-validated by
`meta.schema.json`, propagated into `models.json`):

```json
"assembly": {
  "stl": "assembly.stl",
  "parts": [
    { "stl": "lake.stl",    "color": "#64b5f6" },
    { "stl": "terrain.stl", "color": "#9e9e9e" },
    { "stl": "snow.stl",    "color": "#f5f5f5" }
  ]
}
```

`stl` names the project STL whose card this composite replaces; `parts` are
STL/colour pairs that must already be co-registered (share the same origin
and footprint) so they can be loaded into one scene with no offset. Every
viewer computes a `composite` descriptor by matching the card's model
against `projectData.assembly.stl`, re-validating each part's `color` against
`/^#[0-9a-fA-F]{6}$/` as defence-in-depth even though the schema already
enforces it:

- **`index.html` / `embed.html`**: `createViewer()`'s `loadComposite(parts)`
  loads every part STL with `STLLoader`, computes the union bounding box
  across all parts, centers the whole group at the origin (mirroring
  `geometry.center()` for a single model), and adds one `THREE.Mesh` per
  part — each with its own `MeshPhongMaterial` — into a `THREE.Group`. The
  cross-section toggle iterates `handle.materials` (one material for a
  single model, one per part for a composite) so clipping still applies to
  every part. A composite card **suppresses the filament color picker**
  entirely, since colours are fixed per part, not user-selectable.
- **Standalone viewers** (`generate-standalone.py`): `build_composite_js()`
  reads the same `assembly` object from the project's `meta.json` at build
  time, resolves each part's STL to a base64 blob, and serialises
  `COMPOSITE_PARTS` into the generated HTML. The standalone JS scene-setup
  mirrors `index.html`'s `loadComposite()` logic and also skips the color
  picker for composite pages.

`nz-ski-fields/assembly.scad` still exists, but only as the **thumbnail
source** — it renders Z-up (no `rotate([-90, 0, 0])`, unlike the printable
parts) from a downsampled 128px heightmap so OpenSCAD's default camera looks
down onto the coloured terrain for the PNG. Its STL output is no longer
loaded by any viewer.

### 3D Controls

Each model card has a row of control buttons that appear below the 3D canvas once
the STL is loaded (disabled until then):

- **Rotate 90°** (`↺ Y` / `↺ X`): Rotates the mesh 90° around the world Y or X
  axis. Useful for upright models where OrbitControls' fixed world-up otherwise
  prevents free rotation. After rotation the cross-section clip bounds are
  recomputed.
- **Reset**: Restores the camera to its default position and re-initialises the
  active control mode.
- **Orbit / Trackball / Arcball**: Three toggle buttons switch the active orbit
  style at runtime. **Orbit** (default, OrbitControls, world-Y-up), **Trackball**
  (TrackballControls, gimbal-lock-free), **Arcball** (ArcballControls with
  on-screen gizmo rings suppressed via `setGizmosVisible(false)`). All three
  classes are imported from `three/addons/` at Three.js 0.170.0 with no extra
  import-map entries.

The `makeControls(mode)` function handles switching: it clones `controls.target`,
calls `controls.dispose()`, instantiates the new mode (calling
`controls.handleResize()` for TrackballControls), then restores the saved target
and calls `controls.update()`. The `resize()` loop calls `controls.handleResize?.()` via optional chaining so the TrackballControls screen-rect cache stays current. The active mode button uses `aria-pressed="true"` to reflect state.

### Cross-Section View

Each model card has a "✂ Cross Section" toggle button that clips the rendered
model with a horizontal plane, revealing interior geometry. A depth slider
controls the clip plane position along the model's Z axis. Pressing `C` toggles
the feature via keyboard. Implemented using Three.js local clipping
(`renderer.localClippingEnabled = true`) with a `THREE.Plane` applied to the
model material. Available in `index.html`, `embed.html`, and standalone viewers.

### Maximize Preview

Each model card has a maximize button (shown on hover, below the fullscreen
button) that expands the card to a viewport-filling overlay without using the
native browser Fullscreen API. `toggleMaximize(card)` adds `.card--maximized`
to the card and `.has-maximized` to `<body>` (disabling page scroll), and
inserts a `.maximize-backdrop` div behind the card. Clicking the backdrop
dismisses the overlay. Pressing `M` toggles maximize for the focused card;
`Escape` dismisses a maximized card.

Z-index hierarchy: backdrop = 40, maximized card = 50, QR popover = 100 (QR
remains above the overlay). The `resize()` loop inside `createViewer` runs
every animation frame checking `canvas.clientWidth/clientHeight`, so the
renderer self-corrects when CSS changes the card's dimensions — no explicit
resize trigger is needed. The fullscreen button guard exits maximize state
before calling `requestFullscreen()` to prevent stacking both modes.

### Deep Links (URL Hash Routing)

Models are addressable via URL hash in the format `#<project-slug>/<model-slug>`
(e.g., `#power-workshop/drill-bit`). The `slugify()` function normalizes
project display names and STL filenames into URL-safe hyphenated slugs.
Cards and sections have `data-slug` attributes for O(1) lookup.
`navigateToHash()` expands collapsed sections, activates the viewer, and
scrolls to the target card. Hash updates use `replaceState` (not `pushState`)
to avoid polluting browser history. The `hashchange` event handles
back/forward navigation.

### QR Codes

Each model card has a QR button (top-left, shown on hover) that opens a
centered popover with a QR code image linking to the model's deep link URL.
QR PNGs are generated during CI by `qrencode` and stored in `site/qr/`. The
QR images use the site's dark theme colors (`foreground: #E0E0E0`, `background:
#1A1A2E`). The popover is dismissed by clicking outside or pressing Escape.
The `qr` field in `models.json` is only present when the QR file exists,
so the button degrades gracefully.

### Touch Gesture Hints (Mobile)

On the first touch interaction on a mobile device, a semi-transparent overlay
appears showing three gesture icons (Rotate, Zoom, Pan) with inline SVGs. The
overlay uses `pointer-events: none` so it doesn't block OrbitControls input.
It fades out after 5 seconds or on the first `touchmove`, whichever comes
first. The hint is shown once per session (`sessionStorage`).

### Accessibility

- **`prefers-reduced-motion`**: a CSS media query disables decorative CSS
  transitions; WebGL OrbitControls are user-driven and intentionally unaffected
- **ARIA attributes**: project sections have `role="group"` and `aria-label`;
  section headings have `role="button"`, `tabindex="0"`, and `aria-expanded`;
  canvases have `role="img"` and descriptive `aria-label`; fullscreen button,
  QR button, color swatches, and download/source links all have `aria-label`
  attributes
- **Focus styles**: `:focus-visible` outlines on cards, headings, buttons,
  swatches, and links; `:focus-within` makes the fullscreen and QR buttons
  visible when any card child is focused
- **Keyboard navigation**: Arrow keys move focus between visible cards (skips
  collapsed sections). `F` toggles fullscreen, `D` triggers download,
  `C` toggles cross-section view, `M` toggles maximize preview,
  `R` triggers the customizer Re-render button (when one is present).
  `Escape` dismisses a maximized card (or the QR popover).
  Enter/Space toggles section expand/collapse on focused headings. Modifier
  keys and input fields are excluded from shortcuts.

## Embed Viewer (embed.html)

A minimal single-model 3D viewer designed for iframe embedding via OEmbed.
Reads `#<project-slug>/<model-slug>` from the URL hash, resolves it against
`models.json`, and renders the model full-viewport with the same Three.js
scene setup as `index.html` (background `#0f3460`, `MeshPhongMaterial` blue,
3-light rig, OrbitControls with damping). Includes a small "Open in viewer"
link (bottom-right) that opens the full gallery. Has `<meta name="robots"
content="noindex">` to avoid search indexing. A 10-second timeout detects
CDN load failures and shows a fallback error message. Pauses rendering when
the tab is hidden via `visibilitychange`.

## Standalone HTML Viewers

CI generates one self-contained HTML file per model at `site/standalone/`.
Each file inlines Three.js (as base64 data URIs in the import map) and the
STL data (base64-encoded), requiring **no external dependencies** — works
from `file://` or any static host. Generated by `scripts/generate-standalone.py`,
which downloads Three.js 0.170.0 once, verifies SHA-256 hashes, caches
assets locally (`.cache/threejs/`), and uses the same scene setup (colors,
lighting, camera) as the main viewer. Includes a filament color picker and
fullscreen button.

## Public Source Links

Every model card in `index.html`, the "Open in viewer" overlay in
`embed.html`, and the footer of each standalone viewer link back to the
model's `.scad` source in the **public mirror** at
`https://github.com/stjohnb/3d-models` (not the private
`St-John-Software/3d-models` repo), e.g.
`https://github.com/stjohnb/3d-models/blob/main/power-workshop/drill_bit.scad`.
Each project header in `index.html` also gets a "View on GitHub" link to its
source directory (`tree/main/<dir>`). `index.html` and `embed.html` derive the
link from `models.json`'s `files[].source`; `generate-standalone.py` derives
it from `site/.scad-map`'s third (source-path) column. Source paths are
percent-encoded per path segment (never the `/`) since a small number of
`.scad` filenames contain spaces.

**Invariant**: the `PUBLIC_REPO` constant and `publicSourceUrl()` helper must
stay textually identical across `index.html`, `embed.html`, and
`scripts/oembed_helpers.py` (`PUBLIC_REPO_URL` / `public_source_url()`) — the
same class of three-location sync rule as `slugify()`.

## OEmbed Endpoints

Static OEmbed JSON files are generated per model at
`site/oembed/<project-slug>/<model-slug>.json`. Each is a standard OEmbed
v1.0 "rich" type response with an `<iframe>` snippet pointing to
`embed.html#<project>/<model>`. The iframe uses `sandbox="allow-scripts
allow-same-origin"` for security. `<link rel="alternate">` discovery tags
are injected into `index.html`, but since all tags live in a single page,
auto-discovery only works for the site root — not for individual model deep
links (hash fragments aren't sent to the server). For per-model OEmbed,
consuming platforms should be configured with explicit endpoint URLs.

## Analytics

`index.html` and `embed.html` load a self-hosted Plausible script with the `file-downloads`, `hash`, `outbound-links`, `pageview-props`, and `tagged-events` extensions. The `hash` extension makes the viewer's `#project-slug/model-slug` deep links register as distinct pageviews. The `file-downloads` extension auto-tracks STL and zip download clicks. Standalone viewers (`site/standalone/`) intentionally omit the analytics script to remain self-contained and work offline.
